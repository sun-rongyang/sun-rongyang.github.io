Simple Blog Built by Pelican with blue-penguin theme

Install Python

We use the Anaconda to manage Python version and packages. So we install the Anaconda first.

For the Mac OS X system, we follow the official installation documentation at here. To simplify, you can choose graphical install.

After Install the Anaconda, start the Anaconda-Navigator application. Then start a enviroment at the terminal by Environments -> base(root) -> Open terminal.

Install Pelican

It is easy to install Pelican using pip. At the command line, use the command below

> pip install pelican

where > is the prompt. Since we use the Markdown to write the blog article. We also need install the Markdown package.

> pip install markdown

Create a Pelican project

After the Pelican is installed, we can create our own website project using it. Using the command line, we navigate to the project root path. We substitute the project root path with proj-root.

> cd proj-root

Then use the pelican-quickstart command to create a Pelican project. This interactive command will ask you some question and setup basic things for you. This process seems like

> pelican-quickstart

After these Q&A, Pelican creates setting files, content folder and output folder. we show the basic file structure of a Pelican project here.

.
|-- Makefile        # generate, clean, pubish, etc. the website using make
|-- content         # content paht for the website
|-- output          # root path for the generated website
|-- pelicanconf.py  # Pelican setting file for development
|-- publishconf.py  # Pelican setting file for publish
`-- tasks.py        # Python API to generate, clean, publish, etc. the website. We will not use them.

Use the blue-penguin theme

To use a Pelican theme, we should create a theme folder under the proj-root:

> mkdir theme

Then, install the blue-penguin theme using git from its project repository:

> git clone https://github.com/jody-frankowski/blue-penguin.git theme/blue-penguin

Finally, set the theme to blue-penguin in setting file pelicanconf.py:

THEME = 'themes/blue-penguin'

Do some basic settings

Up to now, we have almost completed. Let us see what we website looks like.

> make devserver

Here, if you just type make, it will display helps to you. Now switch to your web browser, visit 127.0.0.1:8000. You will see something likes

Display tags, categories and archives on menu

Follow the suggestion here: https://github.com/jody-frankowski/blue-penguin, we add the settings to the pelicanconf.py

DISPLAY_PAGES_ON_MENU = False               # Cover default setting, let the blue-penguin to control
TAGS_URL           = 'tags'
TAGS_SAVE_AS       = 'tags/index.html'
CATEGORIES_URL     = 'categories'
CATEGORIES_SAVE_AS = 'categories/index.html'
ARCHIVES_URL       = 'archives'
ARCHIVES_SAVE_AS   = 'archives/index.html'
MENU_INTERNAL_PAGES = (                     # Set pages displayed on the menu
    ('Tags', TAGS_URL, TAGS_SAVE_AS),
    ('Categories', CATEGORIES_URL, CATEGORIES_SAVE_AS),
    ('Archives', ARCHIVES_URL, ARCHIVES_SAVE_AS),
)

Save these settings and Pelican will regenerate the website automatically for you. Now refresh your web browser. You will see

Add an About Me page

Create a markdown file for About Me

> mkdir content/pages
> touch content/pages/aboutme.md

Open the markdown file and write something about yourself like

Title: My Name
Slug: <aboutme-slug>

I am a happy developer. Bla bla ...

Notice that the Slug you used here will be used again later.

In the pelicanconf.py setting file, modify the MENU_INTERNAL_PAGES as

MENU_INTERNAL_PAGES = (
    ('Tags', TAGS_URL, TAGS_SAVE_AS),
    ('Categories', CATEGORIES_URL, CATEGORIES_SAVE_AS),
    ('Archives', ARCHIVES_URL, ARCHIVES_SAVE_AS),
    ('About Me', 'pages/<aboutme-slug>.html', 'pages/<aboutme-slug>.html'),
)

The <aboutme-slug> is the Slug you used in your About Me markdown file.

Now save the setting file and refresh your web browser. You will see

Make your articles more orderly

In principal, you can put your articles to the content directory. But we suggest that you can group all your articles to content/articles using the setting in the pelicanconf.py

ARTICLE_PATHS = ['articles']
STATIC_PATHS = ['articles']

Now the Pelican only process the files in the content/articles.

You can also set the article's url and location by adding

ARTICLE_SAVE_AS = '{date:%Y}/{slug}.html'
ARTICLE_URL = '{date:%Y}/{slug}.html'

to the pelicanconf.py. Now the Pelican will generate your article to <output-dir>/<year>/<slug>.html.

Publish your blog to Github

Generate website

> make html

Create the Github Pages repository on the Github

From now on, consider your github account name is <gh-name>. Visit https://github.com/<gh-name> and create a new public repository called <gh-name>.github.io like

  • step one
  • step two

Push your blog website to the Github Pages repository

Go to the Pelican output directory and initialize a local git repository

> cd output
> git init

Then add all the things in this folder to the git repository

> git add .

Then create a git commit

> git commit -m "My blog's first commit"

Finally, you can push your local git repository to the Github Pages repository you created above

> git remote add origin https://github.com/<gh-name>/<gh-name>.github.io.git
> git push -u origin master

View your blog website

Wait several minutes and go to <gh-name>.github.io to check your website.

Continue to update your blog

After you do some update for your blog, for example you modify a post or write a new one, you can update your website using the following steps.

  • generate your website
> make html
  • update your local git repository
> cd output
> git add .
> git commit -m "<your comments like 'blog update'>"
  • push to remote Github Pages repository
> git push origin master
